RTS 项目介绍

KB 与 Runtime Projection

KB 负责治理完整规则知识，runtime projection 负责把已批准内容发布成服务可读取的运行包。

KB 与投影的分工

KB 保留完整治理，runtime projection 提供运行时视图。

这是 RTS 里最重要的边界：KB 不是给人看的长文档集合，projection 也不是简单摘要。两者都是机器优先结构，只是承担的职责不同。

KB 关注“规则为什么可信”

source、evidence、review、ambiguity、adjudication、signoff 和 lineage。

Projection 关注“服务如何读取”

release、scope、object manifest、L2、cards、dependency、governance summary 和 trace。

01

源材料 Source Material

业务文档、Java/XSLT、Excel mapping、lookup table、样例和澄清记录。

02

KB Knowledge Base

完整规则知识图：rule、lookup、helper、evidence、review、adjudication、signoff。

03

运行时投影 Runtime Projection

已批准内容的运行包：release manifest、scope registry、object manifest、cards、L2、dependency。

04

服务读取 Service Read

查询、索引、API、影响分析和 AI 辅助都通过 projection 读取规则。

05

复盘与演进 Trace / Feedback

调用 trace、review feedback 和新裁决回到治理流程，再发布新 projection。

KB Layer

KB 里应该保留什么？

KB 是完整 governed truth graph，不只是规则正文。它需要保存规则本身，也要保存规则为什么可信、哪里还有歧义、谁做了裁决。

  • rule / lookup / helper
  • source material
  • evidence chain
  • review notes
  • ambiguity / open question
  • human adjudication / signoff

Runtime Projection

Projection 里应该发布什么？

Projection 是 approved truth 的服务运行视图。它应该降低运行时噪声，但不能把规则事实压成只有自然语言摘要。

  • release manifest
  • scope registry
  • object manifest
  • object cards
  • L2 runtime objects
  • dependency edges
  • governance summaries
  • caller profiles

为什么不直接用搜索索引？

索引能帮忙找到对象，但不能成为规则事实。

Object card、L0/L1、alias、confusable 和全文索引适合召回和消歧。真正回答“规则怎么生成”时，服务必须回到 L2 runtime object、dependency 和授权 governance view。

先确定 scope 用 card 找对象 读取 L2 规则正文 展开 dependency 记录 trace

Glossary · 术语解释

KB 与投影里的关键英文词

这些词对应 runtime projection 包里的真实区域或字段。

RTS Rule Truth Source / Transformation Rule System

RTS 将分散的转换规则整理成可治理、可发布、可查询的规则知识基础。

KB Knowledge Base，规则知识库

保存规则逻辑、来源证据、review 结论、歧义和 signoff 的完整治理层。

Rule Pack 规则包

按业务范围组织的一组 rule、lookup、helper，是整理、review 和发布的基本单位。

Runtime Projection 运行时投影

把已批准的 KB 内容发布成服务可读取的运行包，让查询、索引和后续消费有稳定输入。

L2 Runtime Object L2 运行时规则对象

runtime projection 中最重要的事实正文，保存服务可读的结构化规则逻辑。

Object Card 对象卡片

用于定位、消歧和召回规则对象；它帮助找到 L2，但不能替代 L2。

Release Manifest 发布清单

说明当前 projection 来自哪里、包含什么、是否可用，以及出现问题时如何回滚。

Scope 业务范围

通常由 channel、product、pack、domain 组成，用来防止相似规则被串用。

Dependency 规则依赖

描述 rule、lookup、helper 之间的关系，是解释影响面和测试范围的基础。

Governance View 治理视图

在权限允许时展示 evidence、review、open question、risk 和 signoff 信息。

Trace 审计轨迹

记录一次查询或调用读了哪些对象、如何命中、如何拒答，便于复盘和审计。

Index Document 索引文档

从 projection 派生出的搜索和导航视图，只负责召回，不拥有最终规则真相。

AI-assisted Consumer AI 辅助消费方

可以基于 projection 做解释、影响分析或辅助 review，但基础仍是 KB 和 runtime projection。

边界表

哪些内容负责治理，哪些内容负责运行？

内容主要用途介绍时怎么说
KB canonical objects保存完整规则和治理上下文这是规则知识资产本体
Runtime projection发布给服务读取的运行包这是已批准规则的服务视图
L2 runtime object保存服务可读的结构化规则事实回答事实时要回到这里
Object card / L0 / L1帮助搜索、导航和消歧它是入口，不是最终规则正文
Governance summary解释可信度、风险和 open question需要权限和场景控制
Trace复盘一次服务调用读了什么它证明过程，不反写规则真相
