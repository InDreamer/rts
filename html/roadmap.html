路线图

把 confirmed baseline、Day1、Day2、internal agent 和最终服务形态串成一条实现路径。

Baseline

Constitution + Projection Contract

先锁住 truth ownership、runtime projection、refusal、permissioned governance view。

Day1

Query / Tool Service + Controlled Harness

建立文件系统 store、Lucene、L2 read/hash、trace/refusal、REST/MCP 和第一版 /ask。

Internal Agent

Service-owned Managed Loop

planner、orchestrator、context snapshot、claim validation、scenario compiler、evaluation gate。

Day2

Controlled Agentic Retrieval

多步 retrieval、confusable、rerank、expanded MCP、impact/test candidate 和评估闭环。

Final

LLM-agent-ready Rule Truth Service

managed mode 与 external tool mode 共享稳定原子能力和同一 truth access boundary。

现在不该做什么

不要把 OpenSearch、vector、外部 agent SDK 或 memory 写成 truth core；它们最多是 adapter、检索增强或 workflow sidecar。

下一步真正重要什么

不是让模型更会说，而是让 managed analysis 在 /ask 和 scenario endpoint 中成为稳定正常态，同时 claim validation 和 trace 能证明它没有越界。
