The Story

RTS 的核心不是“把规则放进 AI 问答”。

它的目标是让规则从 source material 走到可治理、可追溯、可审查、可查询、可解释的服务真相。

Source

多源材料

业务文档、Java/XSLT、Excel mapping、lookup、示例和 reviewer clarification。

Govern

AI-first review

AI 做结构校验、冲突发现、证据整理和问题简化；人工做最终裁决。

Publish

Runtime projection

approved truth 被发布成服务可读取的 projection，而不是直接读工作区文档。

Serve

API / MCP / Ask

所有调用都经过 release、scope、permission、L2 hash、grounding、trace gate。

Dual Core

两个核心能力，不是主次关系。

确定性能力是第一等公民，托管 LLM 分析也是第一等公民。LLM 不拥有 truth，但它是 RTS 正常产品形态中的受控分析与表达层。
