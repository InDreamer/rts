运行与调用

把启动、验证、API 调用和常见误区放在一条可执行路径里。

01

本地启动

默认使用 sample runtime projection store。服务不直接读取 KB/candidate pack 作为 truth。

RTS_STORE_ROOT="$PWD/sample-projection/runtime-store" \
JAVA_HOME=$(/usr/libexec/java_home -v 17) \
mvn spring-boot:run
02

快速验证

先看 MCP 工具清单，再用当前 active release 的 photo scope 查询。

curl -s http://localhost:8080/mcp/tools | jq
03

提交前检查

运行测试和 diff 检查，避免把运行配置或格式问题带进提交。

JAVA_HOME=$(/usr/libexec/java_home -v 17) mvn verify
git diff --check
git status --short

Try It

选择场景，生成可复制请求。

这个示例生成器是为了降低调用门槛。它不会修改服务状态，也不会替代 API guide。

调用时最容易踩的坑

  • scope 不完整：channel/product/pack/domain 必须属于当前 active release。
  • 字段名写错：请求 body 使用 snake_case，不要用 camelCase。
  • LLM 不可用：/ask 会降级为结构化信息提供服务，不应被描述成等价 managed analysis。
  • candidate 被误用：scenario 输出不是 release approval、final root cause 或 QA signoff。
