Transformation Rule System / OV Boundary

Truth-source ownership stays inside the bank XML transformation rule system. OpenViking is allowed to act as a projection, retrieval, and context-loading shell, but not as the canonical authority.
Truth-source side
  • Canonical packs own banking transformation truth.
  • Evidence, review, ambiguity, and signoff remain governed here.
  • Completed packs become goldensource for their scope.
Boundary rule
  • One-way projection only.
  • Runtime memory never rewrites canonical YAML.
  • OV may retrieve truth, but may not decide truth.
Useful OV pieces
  • Stable addressability and low-noise navigation.
  • L0/L1/L2 reading model for scope-first retrieval.
  • Optional lightweight dependency links later if retrieval needs them.
