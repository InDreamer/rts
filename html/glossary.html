RTS 项目介绍

术语表

RTS 项目中反复出现的英文技术名词。英文保留是为了和真实文件名、字段名、API 接口保持一致。

Glossary · 核心术语

KB、Projection 与服务层

这些词对应 runtime projection 包里的真实区域或字段，会出现在 API 请求、响应和 trace 中。

RTS Rule Truth Source / Transformation Rule System

将分散的转换规则整理成可发布、可追溯、可查询的业务规则知识库，并提供内置 LLM 分析能力。

KB Knowledge Base，业务规则知识库

保存规则逻辑、来源证据、review 结论、歧义和 signoff 的完整知识层。

Rule Pack 规则包

按业务范围组织的一组 rule、lookup、helper，是整理、review 和发布的基本单位。

Runtime Projection 运行时投影

把已批准的 KB 内容发布成服务可读取的运行包，让查询、索引和后续消费有稳定输入。

Release Manifest 发布清单

说明当前 projection 来自哪里、包含什么、是否可用，以及出现问题时如何回滚。

Scope 业务范围

通常由 channel、product、pack、domain 组成，用来防止相似规则被串用。

L2 Runtime Object L2 运行时规则对象

runtime projection 中最重要的事实正文，保存服务可读的结构化规则逻辑。事实回答必须回到这里。

Object Card 对象卡片

用于定位、消歧和召回规则对象；它帮助找到 L2，但不能替代 L2。

Dependency 规则依赖

描述 rule、lookup、helper 之间的关系，是解释影响面和测试范围的基础。

Index Document 索引文档

从 projection 派生出的搜索和导航视图，负责召回定位，规则权威仍在 L2。

Governance View 治理视图

在权限允许时展示 evidence、review、open question、risk 和 signoff 信息。

Trace / Query Trace 审计轨迹 / 查询轨迹

记录一次查询或调用读了哪些对象、如何命中、scope、dependency、grounding 和拒答原因，便于复盘和审计。

Grounding 事实落点

把回答中的事实声明绑定回 L2、dependency、source 或授权治理材料，确保回答有据可查。

Managed Analysis 内置 LLM 分析

RTS 内部 LLM 在服务边界内调用受控工具，生成规则解释、影响评估、测试建议和调查路径。

AI-assisted Consumer LLM / Agent 消费方

可以基于 projection 做解释、影响分析或辅助 review，但基础仍是 KB 和 runtime projection。
