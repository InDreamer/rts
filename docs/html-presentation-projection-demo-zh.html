HTML 只是展示投影，不是 RTS 运行时真相

这个页面演示 HTML 应该长什么样：它可以把 projection 中的规则、证据、风险、AI 分析组织成清晰的审阅界面； 但 runtime、tool、query 和 grounding 不能依赖解析这个页面。真正的 truth 仍来自 JSON/JSONL runtime projection 和 L2 runtime object。

页面身份：这是由 projection 派生的 presentation view。它可以被 AI 或人阅读，但它不是 source of truth。

禁止用法：不要让 QueryService、MCP tool、LLM grounding、release gate 或 dependency traversal 解析 HTML DOM 来获得事实。

HTML 可以展示，但不能接管 truth

推荐用法

由 RTS 或 publisher 从 projection 生成 HTML 报告。

页面中只展示 compact explanation、风险、引用、候选建议和 trace link。

每个展示块都带 uri、release_id、content_hash，方便 AI 或人回查。

禁止用法

不要让 runtime query 通过 CSS selector 或 DOM text 抽取规则事实。

不要让 LLM 把 HTML 页面中的解释文字当成 L2 fact。

不要把人工或 AI 在页面上的批注直接写回 canonical truth。

格式责任对比

Artifact 能做什么 不能做什么
L2 JSON 承载 service-readable structured rule truth，可 hash、可 schema validation、可 grounding。 不负责好看，不直接面向浏览器审阅体验。
Object Card JSONL 帮助 resolver 定位、消歧、召回和排序。 不能替代 L2 成为最终事实来源。
HTML Presentation 展示 AI 分析、对比、风险、引用和少量人工抽查界面。 不能被 runtime/tool/query 当成 truth source 或权限绕过入口。
